# MVP Test Plan

Этот файл закрывает шаг 6 из алгоритма: тестирование на группе пользователей.

## Роли в тесте

- 1 организатор
- 3-5 участников

## Сценарий проверки

1. Авторизация
- зарегистрировать организатора и участников
- проверить вход под каждой ролью

2. Создание контента
- организатор создает квиз
- добавляет минимум 3 вопроса разных типов:
  - текстовый
  - с изображением
  - single/multiple choice

3. Запуск сессии
- организатор запускает комнату и получает room code
- участники подключаются по коду
- организатор стартует квиз и переключает вопросы

4. Ограничение по времени
- проверить, что после окончания таймера ответ отправить нельзя
- проверить, что ответ принимается только один раз на вопрос

5. Подсчет баллов
- проверить начисление очков за правильные ответы
- сверить лидерборд после каждого вопроса
- проверить финального победителя после завершения

6. История
- участник: в истории отображаются баллы и место
- организатор: в истории отображаются проведенные сессии и число участников

## Критерии приемки

- все ключевые функции ТЗ проходят без блокирующих ошибок
- real-time синхронизация между вкладками работает
- лидерборд корректно обновляется и сохраняется
